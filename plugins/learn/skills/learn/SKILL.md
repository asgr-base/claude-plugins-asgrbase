# /learn — セッション学習・設定改善スキル

セッション中の発見・問題・訂正を分析し、プロジェクト設定への改善提案を生成する。  
「また同じ説明をする」「同じ失敗を繰り返す」を防ぐことが目的。

## クイックリファレンス

| モード | 用途 |
|--------|------|
| `/learn` | フル分析 → 提案 → 承認 → 適用 |
| `/learn quick` | 3行サマリーのみ（ファイル変更なし） |
| `/learn apply` | 提案後、確認なしで即適用 |

---

## Phase 1: セッション分析

会話履歴（コンテキストウィンドウ内）を振り返り、以下を抽出する。  
**外部ファイル読み込み・シェルコマンド不使用。OS非依存。**

| カテゴリ | 検出パターン |
|---------|------------|
| **ユーザー訂正** | 「いや」「そうじゃなくて」「やり直して」「no,」「actually」「違う」 |
| **問題と対策** | エラー発生 → 回避策発見 → 解決 のシーケンス |
| **繰り返しパターン** | 同一操作を3回以上手動実行した |
| **発見・示唆** | 「〜だったのか」という理解の更新、新しいツール仕様の把握 |
| **成功パターン** | 「完璧」「ちょうどいい」「まさにそれ」と評価された手法 |
| **前提確認漏れ** | 途中で判明した「最初に確認すべきだった情報」 |

### `/learn quick` はここで終了

3行サマリーを出力して終了する:
```
## セッションサマリー（quick）
1. [最も重要な発見・問題]
2. [2番目に重要な発見]
3. [次回に活かすべきこと]
```

---

## Phase 2: 改善対象の探索

Glob / Read ツールで以下を探索する。**提案が必要なファイルだけ Read すること（全件読み込み禁止）。**

### Step 2-1: CLAUDE.md を探す

```
Glob: CLAUDE.md                   # プロジェクトルート
Glob: **/CLAUDE.md                 # サブフォルダ（全階層）
```

発見した全 CLAUDE.md をリストアップし、各提案の改善先として候補に加える。

### Step 2-2: .claude/ 配下を探す

```
Glob: .claude/rules/*.md           # ルールファイル
Glob: .claude/references/*.md      # チームknowledgeファイル
Glob: .claude/skills/*/SKILL.md    # スキルファイル
Glob: .claude/hooks/**/*           # hooks設定
```

Phase 1 の発見に関連するファイルのみ Read する。

---

## Phase 3: 改善提案の生成

### Step 3-1: 配置先の決定

各発見事項について、**読み込み特性の違い**に基づいて配置先を決める。

| 配置先 | 読み込み | コスト | 向いている内容 |
|--------|---------|--------|--------------|
| `CLAUDE.md` / `rules/` | セッション開始時・必ず | 常に消費 | 短く・毎回守るべきルール（2行以内目安） |
| `references/` | 明示参照された時のみ | 必要時のみ | 長い・条件付きで参照するナレッジ |
| `skills/*/SKILL.md` | スキル呼び出し時のみ | 必要時のみ | スキル実行に特化した手順・知識 |
| Memory | 次回セッション以降 | なし | セッション横断で記憶したい事実 |

**判断フロー（この順に問う）:**

```
Q1. 毎回のセッションで必ず守る必要があるか？
    YES → CLAUDE.md（1-2行）または rules/（3行以上のルール群）
    NO  → Q2 へ

Q2. 特定のスキル実行時にだけ参照するか？
    YES → 該当スキルの SKILL.md または skills 内 references/
    NO  → Q3 へ

Q3. 複数セッションにわたって参照される知識か？
    YES → .claude/references/（※必ず参照ポインタとセット。下記参照）
    NO  → Memory（ファイル変更なし）
```

**references/ に追加する場合は必ず参照ポインタをセットで提案する:**

`references/` はセッション開始時に読まれない。参照ポインタがなければ永遠に読まれない。
以下のいずれかとセットにしなければ意味がない:

- 関連スキルの SKILL.md に `[references/xxx.md](references/xxx.md) を参照` を追加
- または `rules/` ファイルに `詳細は .claude/references/xxx.md を参照` と追記

```
# 例: references/ への提案は常に2件セット
提案A: .claude/references/api-patterns.md を新規作成（ナレッジ本体）
提案B: .claude/rules/tools-debug.md に参照ポインタを追加（読み込みトリガー）
```

### Step 3-2: 提案フォーマット

各発見事項を以下のフォーマットで出力する。  
各対象の書き方基準は [references/targets-guide.md](references/targets-guide.md) を参照。

```
### [番号]. [提案タイトル]
**対象**: [ファイルパス]
**確信度**: 高 / 中 / 低
**根拠**: このセッションで何が起きたか（1-2文）
**提案内容**:
[実際に追記・変更するテキスト]
**期待効果**: 次回セッションでどう改善されるか（1文）
```

### 提案数の上限

| 確信度 | 上限 |
|--------|------|
| 高 | 無制限 |
| 中 | 3件まで |
| 低 | 1件まで |

**過剰提案を避ける**: 1セッションで5件を超えたら高確信度に絞ること。

---

## Phase 4: Checkpoint → 承認 → 適用

### Step 4-1: 提案一覧の出力

全提案を出力後、以下を提示する:

```
適用方法を選択してください:
- すべて適用
- 番号を指定（例: 1, 3）
- キャンセル
```

### Step 4-2: Checkpoint（`/learn apply` はスキップ）

各ファイルの書き込み前に、変更箇所を示して確認を取る:

```
[Checkpoint] CLAUDE.md に以下を追記します:

追記場所: ## HOW（必須ルール） > 操作ルール
追記内容:
---
[提案内容のテキスト]
---
続行しますか？ (y/n)
```

### Step 4-3: 適用

- 既存ファイルへの追記: Edit ツール（既存構造を壊さない位置に挿入）
- 新規ファイルの作成: Write ツール
- 適用後: 変更ファイル一覧を報告

### Step 4-4: 却下された提案の扱い

ユーザーが適用を拒否した場合、以下のどちらかを選択:

1. **Memory に保存**: 将来の参考として記録（ファイル変更なし）
2. **破棄**: 完全に無視

---

## 注意事項

- **根拠なき提案をしない**: 会話履歴に証拠がある発見事項のみ提案する
- **CLAUDE.md を肥大化させない**: 既存セクションで対応できる場合は新セクションを作らない
- **スキル作成は提案に留める**: 実際の作成は `skill-creator` スキルで行うよう案内する
- **hooks 提案は慎重に**: ユーザーの通常作業フローに影響するため、高確信度のみ提案
